# Проект "приложение для интернет-магазина одежды на IOS и Android". 
## Краткое описание:
- Уже есть сайт на wordpress/woocommerce (все данные и функционал будут браться оттуда).
- Базовые стэк - React Native / Node.js / WooCommerce / МойСклад
- Источник данных товаров и фотоконтента - WooCommerce.
- Источник данных о заказах МойСклад.
- Основной ключ идентификации клиента — номер телефона (формат `7xxxxxxxxxx`).
- Все товары вариантивные (вариативность по размеру).
- База данных сайта на отдельном облачном сервере, сайт на основном сервере. Изображения храняться в папке uploads на основном сервере.
- Для оплаты интеграция платежного шлюза robokassa.ru (с возможностью интеграции новых платежных шлюзов).
- Планируется создать VPS-сервер на хостинге beget (без использования docker).

# 0. Общая архитектура проекта

## Источник данных

- Товары, вариации, остатки, цены, пользователи, история заказов → WooCommerce
- После успешной оплаты заказы передаются в МойСклад
- Оплата через Robokassa
- Авторизация по email/паролю и по телефону (VoicePassword)
- Бонусная система через GetMeBack

## Архитектурный принцип

WooCommerce — единственный источник правды (Single Source of Truth).

Мобильное приложение НЕ работает напрямую с WooCommerce.  
Вся логика проходит через собственный backend (Node.js).

WooCommerce — это коммерческий центр системы (e-commerce ядро)
МойСклад — это учётная система (ERP / склад / бухгалтерия)

---

## Логика потока заказа

- Пользователь оформляет заказ в мобильном приложении (корзина → данные доставки и оплаты).
- Backend **сразу создаёт документ «Заказ покупателя»** в МойСклад через JSON API 1.2 (статус — «Новый» / draft).
- Backend возвращает пользователю ссылку на оплату через Робокасса (orderId из МойСклад передаётся как внешний идентификатор).
- Пользователь оплачивает заказ.
- При успешном callback от Робокасса: Backend меняет статус заказа в МойСклад на «Оплачен».
- Приложение получает актуальный статус заказа путём периодического опроса или через webhook от Backend.
- WooCommerce используется **только как источник каталога** (товары, цены, изображения). Создание и хранение заказов полностью вынесено в МойСклад.

---

## Минимальная админ-панель (только заказы и статусы)
- Backend: существующий Node.js + Express.
- Шаблоны: **EJS** (один-два файла).
- Хостинг: та же папка, что и Backend (`/admin`).
- Авторизация по логину/паролю (JWT).
- Источник данных о заказах МойСклад.

# 10. Этап 10 — Оформление заказа

Алгоритм:

1. Backend повторно проверяет остатки
2. Создает заказ в WooCommerce (status = pending)
3. Возвращает order_id
4. Генерирует подпись Robokassa

---

# 11. Этап 11 — Оплата (Robokassa)

Процесс:

1. Открытие WebView
2. Пользователь оплачивает
3. Robokassa отправляет callback
4. Backend:
   - проверяет подпись
   - меняет статус заказа
   - отправляет заказ в МойСклад

⚠ Обязательно защита от повторного callback.

---

# 12. Этап 12 — Передача заказа в МойСклад

После успешной оплаты:

Backend:

1. Создает/находит контрагента
2. Создает заказ
3. Передает:
   - SKU вариации
   - количество
   - цену
   - данные клиента

Фото не передаются.
